# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation — Authentication, RLS, and Audit Trail.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui |
| Preset | to be initialized at project scaffold — run `npx shadcn init` with slate base color |
| Component library | Radix UI (via shadcn/ui) |
| Icon library | lucide-react (bundled with shadcn/ui) |
| Font (heading) | Playfair Display — serif, 700/600 weights — matches pitch website |
| Font (body/UI) | DM Sans — sans-serif, 400/500/600 weights — matches pitch website |
| Font (mono/code) | DM Mono — for audit log values, record IDs, timestamps |

**Source:** eGov-website/index.html design tokens — pitch website uses Playfair Display + DM Sans + DM Mono.
Institutional character is preserved by carrying the same typeface pair into the platform UI.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline element gaps |
| sm | 8px | Input internal padding (vertical), label-to-field gap |
| md | 16px | Default element spacing, card internal padding, form field gap |
| lg | 24px | Card-to-card gap, section padding, form section gap |
| xl | 32px | Page content top padding, layout column gap |
| 2xl | 48px | Major section breaks, auth card vertical padding |
| 3xl | 64px | Page-level breathing room (auth page vertical centering margin) |

Exceptions:
- Touch targets (buttons, checkboxes): minimum 44px height regardless of visual size (WCAG 2.5.5)
- Table row height: 48px minimum for data density comfort on audit log viewer
- Auth card max-width: 420px (fixed, not a spacing token — constrains form width on wide viewports)

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Display heading | 28px | 700 (Playfair Display) | 1.2 | Playfair Display |
| Section heading | 20px | 600 (DM Sans) | 1.3 | DM Sans |
| Body | 16px | 400 (DM Sans) | 1.5 | DM Sans |
| Label / UI text | 14px | 500 (DM Sans) | 1.4 | DM Sans |
| Mono / data | 13px | 400 (DM Mono) | 1.4 | DM Mono |

Typography rules:
- Heading on auth screens uses Playfair Display 28px/700 to convey institutional authority.
- All form labels, button text, table headers, and nav items use DM Sans 14px/500.
- Body copy (instructions, helper text, descriptions) uses DM Sans 16px/400.
- Audit log values (record IDs, JSON diffs, timestamps) use DM Mono 13px/400.
- No more than 4 type sizes in any single screen. Never mix sizes within the same visual group.

## Color

The platform uses a **light-mode paper surface** — the inverse of the pitch website's dark ink theme.
Navy and gold are preserved as brand anchors. Paper (#F3F7FD) becomes the dominant surface color.

| Role | Value | Usage |
|------|-------|-------|
| Dominant surface (60%) | #F3F7FD | Page background, auth page backdrop |
| Secondary surface (30%) | #FFFFFF | Cards, form panels, sidebar, modal backgrounds |
| Nav / app shell | #050D1B | Top navigation bar background (navy — brand anchor) |
| Accent (10%) | #C8A44A | Reserved for specific elements — see list below |
| Destructive | #E74C3C | Destructive action buttons, error borders, error text |

**Accent (#C8A44A) is reserved exclusively for:**
- Primary submit button on auth screens (Sign In, Create Account)
- Active navigation item indicator
- Status badge: "Pending Approval" background tint (gold-pale: #F3E2BC)
- MFA setup "Copy codes" download button
- Table row "Approve" action button in admin user management

**Accent is NOT used for:** hover states, focus rings, links, secondary buttons, or any decorative purpose.

Full token set (maps to CSS variables and Tailwind config):

| CSS Variable | Hex | Tailwind Token |
|---|---|---|
| --c-ink | #050D1B | `navy-950` |
| --c-ink-mid | #0B1625 | `navy-900` |
| --c-paper | #F3F7FD | `paper` |
| --c-paper-line | #D7E2EF | `paper-border` |
| --c-ink-on-paper | #0B1625 | primary text on light bg |
| --c-ink-on-paper-2 | #3A5270 | secondary text on light bg |
| --c-gold | #C8A44A | accent |
| --c-gold-pale | #F3E2BC | accent-pale (badge tint) |
| --c-ok | #27AE60 | success / active badge |
| --c-warn | #E67E22 | warning |
| --c-err | #E74C3C | destructive / error |

**Status semantics used in Phase 1:**
- Pending Approval: gold-pale background (#F3E2BC), navy text — "Awaiting Review"
- Active / Approved: ok-tinted (#DCFCE7 — Tailwind green-100), success text — "Active"
- Suspended: gray background, muted text — "Suspended"
- MFA Required: warn tint (#FEF3C7), warn text — "MFA Required"

**Contrast requirements:**
- Body text (#0B1625) on paper (#F3F7FD): 14.8:1 — exceeds WCAG AAA
- Secondary text (#3A5270) on paper (#F3F7FD): 6.1:1 — meets WCAG AA Large
- Gold accent (#C8A44A) on navy (#050D1B): 5.9:1 — meets WCAG AA for large text / UI
- White text on gold (#C8A44A): 2.4:1 — FAILS WCAG AA. Use navy text on gold backgrounds.
- Navy text (#050D1B) on gold (#C8A44A): 5.9:1 — PASS. Always use --c-ink text on gold backgrounds.
- Error text (#E74C3C) on paper (#F3F7FD): 4.5:1 — meets WCAG AA

## Component Inventory

All components use shadcn/ui primitives. States must be implemented in this order: default → hover → focus → disabled → error → loading.

## Copywriting Contract

### Tone

- **Register:** Formal. Institutional. Direct. Active voice.
- **Avoid:** "Oops", "Uh oh", "Looks like", "Whoops", "Amazing", "Awesome".
- **Use:** "Invalid", "Unable to", "Your account", "Required", "Please verify", "Contact your administrator".
- **Severity language:** Errors are problems to be resolved, not apologies.
- **Action labels:** Always verb + noun. Never noun alone. "Sign In" not "Login". "Create Account" not "Register".

## Interaction Contracts

### Form Validation Timing

| Rule | When triggered |
|---|---|
| Required field validation | On blur (when user leaves field) |
| Format validation (email, password) | On blur |
| Password match validation | On blur of "Confirm password" field |
| Password strength indicator | On every keystroke (real-time, visual only — not an error state) |
| Cross-field validation (server-side duplicates) | On submit only |
| MFA code validation | On submit (or auto-submit on 6th digit for TOTP) |
| All fields re-validated | On submit, before API call |

**No live validation on first interaction.** Field shows error only after user has touched it (onBlur) or attempted submit.

### Loading States

| Action | Loading behavior |
|---|---|
| Sign In submit | Button shows spinner, text "Sign In" remains, button disabled, no page shift |
| Register submit | Button shows spinner, text "Request Access" remains, button disabled |
| MFA verify | Button shows spinner, OTP boxes become read-only |
| Approve user | Approve button shows spinner, row grays out, other row actions disabled |
| Reject user | Reject button shows spinner, dialog confirm button disabled |
| Export CSV | Export button shows spinner, "Exporting..." text, button disabled |
| Audit log filter apply | Table shows skeleton rows (3 rows, gray shimmer), filter bar input disabled |
| Page navigation | Skeleton rows appear immediately on page change |

Skeleton rows: same height as real rows (48px), gray shimmer animation (left-to-right gradient sweep), neutral bg.

### Success Transitions

| Action | Success behavior |
|---|---|
| Sign in (single role) | Redirect to /dashboard, no toast (destination page loads) |
| Sign in (multiple roles) | Redirect to /auth/select-role |
| Role selected | Redirect to /dashboard |
| Register | Redirect to /register/pending |
| MFA setup complete | Redirect to /dashboard, success toast: "MFA enabled. Your account is now secured." |
| MFA challenge passed | Redirect to /dashboard or originally requested URL |
| User approved | Table row status updates to Active (no page reload), success toast: "[Name] has been approved." |
| User rejected | Table row removed with fade-out animation, info toast: "Account request rejected." |
| User suspended | Status badge updates to Suspended in place, info toast: "[Name]'s account has been suspended." |
| Backup codes copied | Toast: "Backup codes copied to clipboard." (4s) |
| Backup codes downloaded | Toast: "Backup codes downloaded." (4s) |
| Audit log exported | Toast: "Export complete. Your file has been downloaded." |

### Error Feedback Patterns

- **Inline field errors:** Red text (14px/400 #E74C3C) directly below the field, no icon, mt-4px.
- **Form-level errors:** Alert (destructive variant) above the submit button — used for errors that span fields (e.g., "Email or password incorrect" — never reveal which one specifically).
- **Network errors:** Toast (persistent, destructive) — must be manually dismissed.
- **Rate limit:** Form-level alert with countdown: "Please wait [N] seconds before trying again."
- **MFA shake animation:** 3-cycle horizontal shake (4px, 120ms total) on invalid code submit. Input border turns red.

### Keyboard Navigation

- All interactive elements reachable via Tab key in DOM order.
- Tab order on auth forms: logo area (skip link) → email → password → submit → secondary links.
- Role selection screen: Tab between role cards, Enter/Space to select, Enter on focused card to proceed.
- OTP boxes: Tab advances between boxes. Backspace moves to previous box.
- Audit log table: Tab to filter bar inputs, then table rows. Arrow keys within table rows to navigate between action buttons.
- Dialogs: Focus trapped within dialog while open. Escape closes dialog (cancel).
- Toast: Escape dismisses toast. Focus not moved to toast.

## Accessibility

### Focus Management

- Focus ring: 2px solid #C8A44A, 2px offset — visible on all interactive elements, including inside auth cards.
- Focus ring is never suppressed. `outline: none` is never used without an equivalent visible indicator.
- On dialog open: focus moves to first interactive element in dialog.
- On dialog close: focus returns to the trigger element.
- On page navigation (SPA): focus moves to page heading (`<h1>`).
- After form error: focus moves to first invalid field.
- After MFA code submission failure: focus returns to first OTP digit box.

### Color Contrast Summary

All text/background combinations meet WCAG 2.1 AA minimum (4.5:1 for normal text, 3:1 for large text ≥18px or ≥14px bold).

| Combination | Ratio | WCAG level |
|---|---|---|
| #0B1625 on #F3F7FD | 14.8:1 | AAA |
| #0B1625 on #FFFFFF | 16.1:1 | AAA |
| #3A5270 on #F3F7FD | 6.1:1 | AA |
| #050D1B on #C8A44A | 5.9:1 | AA (large text/UI) |
| #FFFFFF on #050D1B | 19.2:1 | AAA |
| #E74C3C on #FFFFFF | 4.5:1 | AA |

| #92400E on #FEF3C7 | 5.8:1 | AA |
| #1D4ED8 on #EFF6FF | 5.4:1 | AA |

**Do not use:** gold text (#C8A44A) on white (#FFFFFF) — ratio 2.4:1, fails WCAG AA. Always use navy text on gold backgrounds.

### Motion and Animation

- Transitions: max 320ms for UI transitions. Use `prefers-reduced-motion` media query.
- When `prefers-reduced-motion: reduce`: disable shake animations, disable fade transitions, use instant state changes.
- Loading spinners: rotational only (no translate). Acceptable under reduced motion as they communicate state.
- Skeleton shimmer: replaced with static gray bg when `prefers-reduced-motion: reduce`.
